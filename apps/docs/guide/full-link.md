# 前后端联调主线

如果你要最快理解整个项目，最推荐抓住下面这条主线：登录 -> 获取用户信息 -> 获取菜单 -> 动态路由 -> 进入业务页。

## 1. 登录主线

### 第一步：前端提交登录表单

前端登录页会收集：

- 用户名
- 密码
- 验证码
- uuid

然后调用后端登录接口。

### 第二步：后端校验登录

后端会依次处理：

1. 判断是否开启验证码
2. 去 Redis 校验验证码
3. 校验用户名和密码
4. 校验用户是否被禁用或停用
5. 生成 token
6. 把完整登录态写入 Redis

### 第三步：前端保存 token

登录成功后，前端会把 token 存起来，为后续请求自动带上 `Authorization`。

## 2. 获取当前用户信息

前端拿到 token 后，会调用“获取当前登录用户信息”接口。

这个接口的作用是让前端知道：

- 当前是谁
- 当前用户 ID 是多少
- 当前用户名和头像是什么

这一步完成后，前端才算真正拿到登录上下文。

## 3. 获取菜单并生成路由

这是后台项目区别于普通网站的关键。

### 3.1 后端做什么

后端会根据：

- 当前用户
- 当前用户角色
- 角色绑定菜单

返回一份菜单树。

### 3.2 前端做什么

前端收到菜单树后会：

1. 遍历菜单
2. 将 `component` 字段映射到 `views/**/*.vue`
3. 使用 `router.addRoute()` 动态注入路由
4. 生成左侧菜单

因此，菜单接口成功并不代表页面一定能正常打开，还要保证组件路径能对得上。

## 4. 进入业务页面后的联调模式

进入任意 CRUD 页面后，联调通常会进入下面这个固定节奏：

1. 列表查询
2. 筛选与分页
3. 打开新增弹窗
4. 提交表单
5. 刷新列表
6. 编辑、删除、导出等操作

这类页面在本项目里很多，模式相对统一。

## 5. 上传能力的联调方式

如果页面涉及图片或附件上传，通常流程是：

1. 前端选择文件
2. 调用上传接口
3. 后端保存本地或上传到云存储
4. 后端返回文件 URL
5. 前端保存 URL 并展示预览

## 6. 权限是如何前后配合的

项目权限不是只在前端做，也不是只在后端做，而是两边共同生效。

### 6.1 前端侧

- 是否显示菜单
- 是否允许进入对应路由
- 是否显示某些按钮

### 6.2 后端侧

- 是否允许访问接口
- 当前角色是否满足 `RequireRole`
- 当前用户的数据范围是否符合要求

所以即使前端显示了按钮，后端权限不通过，接口依然会被拦截。

## 7. 你排查联调问题时该怎么想

推荐按下面顺序查：

1. 页面是否拿到了 token
2. 请求头里是否带了 `Authorization`
3. 当前用户信息接口是否成功
4. 菜单接口是否成功
5. 菜单 `component` 是否能匹配到真实页面文件
6. 后端接口是否被守卫拦截
7. 数据库与 Redis 是否可用

## 8. 最重要的结论

对这个项目来说，真正的核心联调路径不是“打开某个列表页”，而是：

- 登录打通
- 菜单打通
- 权限打通

这三件事稳定了，后续新增业务模块会顺很多。
