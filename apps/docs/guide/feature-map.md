# 核心功能地图

这一章用功能视角，而不是代码视角，帮助你快速知道项目已经具备哪些核心能力。

## 1. 平台基础能力

- 用户登录
- 验证码
- token 刷新
- 当前用户信息获取
- 动态菜单
- 动态路由

## 2. 系统管理能力

- 用户管理
- 角色管理
- 菜单管理
- 部门管理
- 岗位管理
- 参数配置
- 公告管理
- 个人中心

## 3. 监控能力

- 登录日志
- 操作日志
- 缓存监控
- 服务监控

## 4. 基础设施能力

- Redis 封装
- Swagger 文档
- 文件上传
- 分片上传
- Bull 队列
- 定时任务
- 邮件推送
- MQTT 插件能力

## 5. 前端基础能力

- 后台布局系统
- 标签页管理
- 全局组件
- 全局指令
- 通用 hooks
- 请求重试与 token 刷新
- 表格与表单开发范式

## 6. 最值得优先理解的功能点

如果你时间有限，优先理解：

1. 登录
2. 菜单
3. 权限
4. 用户与角色
5. 上传
6. 监控页面

## 7. 二次开发最常见切入点

### 如果你要新增业务功能

优先参考这些模式：

- 系统管理类 CRUD 页面
- 后端 `system/*` 模块写法
- 前端 `views/system/*` 页面写法

### 如果你要改平台能力

优先参考这些地方：

- 登录：`modules/main`、`store/auth`、请求封装
- 权限：菜单服务、路由守卫、权限 store
- 上传：`modules/upload`
- 任务：`modules/tasks`

## 8. 这份地图怎么用

建议你在阅读源码前先记住两点：

- 这个项目不是只有 CRUD，它已经带了比较完整的平台基础设施
- 这个项目的大部分复杂度都集中在登录、菜单、权限和基础设施层
